# Theming

Strait's appearance is driven entirely by CSS custom properties. Every colour,
radius, surface, shadow, duration, and stacking layer is a semantic token;
components reference the token, never a raw value. Theming ‚Äî including dark
mode ‚Äî is therefore a matter of redefining a handful of variables.

## Token architecture

Tokens live in two layers in `globals.css`:

1. **`:root`** holds the light-theme values as `oklch` colours.
2. **`.dark`** overrides the same variable names with their dark values.

```css
:root {
  --background: oklch(1 0 0);
  --foreground: oklch(0.216 0.006 56.043);
  --brand: oklch(0.6696 0.222 37.42);
  /* ‚Ä¶surfaces, status intents, charts, radius‚Ä¶ */
}

.dark {
  --background: oklch(0.182 0.004 49.25);
  --foreground: oklch(0.985 0.001 106.423);
  /* same names, dark values */
}
```

A Tailwind `@theme inline` block then maps each token to a utility, so
`--primary` becomes `bg-primary`, `--muted-foreground` becomes
`text-muted-foreground`, and so on. See **Foundations ‚Üí Colors** for the full set.

## Dark mode

Dark mode is a `.dark` class on (or above) the element you want themed ‚Äî
typically `<html>`. Add or remove the class to switch themes; because the tokens
cascade, every Strait component re-themes instantly.

The recommended wiring is [`next-themes`](https://github.com/pacocoursey/next-themes):

```tsx
import { ThemeProvider } from "next-themes";

export function Providers({ children }: { children: React.ReactNode }) {
  return (
    <ThemeProvider attribute="class" defaultTheme="system" enableSystem>
      {children}
    </ThemeProvider>
  );
}
```

Any approach that toggles a `.dark` class works ‚Äî `next-themes` just handles
system preference, persistence, and avoiding a flash on load.

## Rebrand with one colour

`--brand` is the single source of truth for your accent. Set it to **any** colour ‚Äî
hex, `rgb()`, or `oklch()` ‚Äî and the rest of the brand system derives from it
automatically:

```css
@import "@strait/ui/css";

:root {
  --brand: #6366f1; /* that's it ‚Äî derivations follow */
}
```

Setting `--brand` re-themes everything that keys off it:

- **`--brand-foreground`** ‚Äî text drawn on the solid brand fill. Defaults to white
  as a deliberate brand-identity choice. If you rebrand to a light colour that needs
  dark text, override this token.
- **`--brand-accent`** ‚Äî the readable text/border colour used by the soft and outline
  brand variants on tinted surfaces. Darkens in light mode and lightens in dark mode
  to clear WCAG AA on the tint.
- **`--chart-1`**, **`--sidebar-active-rail`**, and the active-row tint all follow
  `--brand` too.

The same value is used in light and dark mode. Each derived token is an ordinary
custom property, so you can still override any of them individually as an escape
hatch ‚Äî and tweak unrelated tokens like radius the same way:

```css
:root {
  --brand: #6366f1;
  --brand-accent: oklch(0.45 0.2 277); /* optional: hand-tune the soft/outline text */
  --radius: 0.5rem; /* round things a little less */
}
```

## Surface tokens

Reusable surfaces are tokenized too, so elevated cards, nested toolbars, and
terminal/code surfaces can be re-themed without chasing component classes:

- **`--surface-raised`** ‚Äî elevated or floating containers that should separate
  from the page background while staying theme-aware.
- **`--surface-subtle`** ‚Äî nested panels, toolbar wells, and low-emphasis
  sections inside cards.
- **`--surface-terminal`** and **`--surface-terminal-foreground`** ‚Äî forced dark
  terminal/code surfaces used by `CodeBlock variant="dark"`.
- **`--overlay`** ‚Äî modal scrims behind dialogs, drawers, and sheets.

Override these alongside your brand when your product needs a flatter, higher
contrast, or more dimensional visual language.

## Elevation, motion & layering tokens

Beyond colour, the system tokenizes elevation, motion, and stacking ‚Äî all
ordinary custom properties on `:root`, overridable the same way as colours:

- **Shadows** ‚Äî `--shadow-sm` (cards at rest), `--shadow-md` (floating
  surfaces, paired with `ring-1 ring-foreground/10`), and `--shadow-lg`
  (hover-lift, modals). Dark mode deepens the alpha; elevation there is
  primarily the `--surface-raised` step plus the hairline ring. See
  **Foundations ‚Üí Shadows**.
- **Motion durations** ‚Äî `--duration-instant` (100ms), `--duration-fast`
  (150ms), `--duration-base` (250ms), `--duration-slow` (400ms), and
  `--duration-deliberate` (600ms), consumed as `duration-(--duration-*)`.
  Exits run one step faster than enters. The `ease-out` / `ease-in` /
  `ease-in-out` utilities map to the system curves. See
  **Foundations ‚Üí Motion**.
- **Z-index layers** ‚Äî `--z-base` through `--z-tooltip`, consumed as
  `z-(--z-*)`, so tooltips always beat toasts, toasts beat modals, and nothing
  wedges between a dialog and its scrim. See **Foundations ‚Üí Layers**.
- **Icon sizes** ‚Äî `--icon-xs` through `--icon-lg`, applied through each
  control's `[&_svg]:size-*` selector. See **Foundations ‚Üí Icons**.

## Status & brand triads

Brand and each status intent come as a triad so you can theme solid, soft, and
outline emphasis consistently:

- **`--<name>`** ‚Äî the solid fill (e.g. a solid button background).
- **`--<name>-foreground`** ‚Äî text/icon colour drawn on top of the solid fill.
- **`--<name>-accent`** ‚Äî a readable text/border colour for soft and outline
  variants on tinted (10%-opacity) surfaces, tuned to clear WCAG AA 4.5:1.

The intents are `success`, `destructive`, `info`, and `warning`; `primary` follows
the `base` / `-foreground` shape. When you override a status intent, keep its triad
in sync so soft and outline variants stay legible.

`brand` carries the same triad, but you only set `--brand` ‚Äî its `-foreground` and
`-accent` derive automatically (see [Rebrand with one colour](#rebrand-with-one-colour)).
