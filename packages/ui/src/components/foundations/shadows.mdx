# Shadows

Elevation uses three tokenized steps — `--shadow-sm`, `--shadow-md`, and
`--shadow-lg` — applied with the `shadow-sm` / `shadow-md` / `shadow-lg`
utilities. Shadows signal layering: resting cards use the smallest step,
floating surfaces (popovers, dropdowns) use `shadow-md`, and hover-lift and
modals use `shadow-lg`.

## Dark-mode elevation

Shadows barely read on dark surfaces. In dark mode, elevation is expressed as
a lighter surface step — `--surface-raised` over `--background` — plus the
standard `ring-1 ring-foreground/10` hairline. The shadow tokens stay applied
with a deepened alpha, but they are the secondary cue, never the only one.

Prefer the smallest shadow that still communicates the layer. Over-elevation
flattens the hierarchy — if everything floats, nothing does.
