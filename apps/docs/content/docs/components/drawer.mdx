## Accessibility

Drawer is powered by the Vaul library, which wraps its content in a dialog-like overlay. The panel traps focus while open and restores focus to the trigger on close. Always include a `DrawerTitle` so screen readers announce the panel purpose when it opens.

### Keyboard interactions

| Key | Action |
| --- | --- |
| `Enter` / `Space` | Opens the drawer when focus is on the trigger |
| `Esc` | Closes the drawer and returns focus to the trigger |
| `Tab` | Moves focus to the next focusable element inside the drawer |
| `Shift` + `Tab` | Moves focus to the previous focusable element inside the drawer |

### ARIA and labeling

`DrawerTitle` and `DrawerDescription` map to Vaul's accessible title/description slots, which are associated with the dialog container via `aria-labelledby` and `aria-describedby`. The drag handle on `direction="bottom"` drawers is decorative; it does not require a label.
