## Accessibility

Built on the `cmdk` library, which implements a fully keyboard-navigable command palette. All filtering, item focus management, and selection are handled internally; no extra wiring is needed. When used inside `CommandDialog`, the dialog title and description are rendered `sr-only` so screen readers announce the palette on open.

| Key | Action |
| --- | --- |
| `Arrow Down` | Move focus to the next item |
| `Arrow Up` | Move focus to the previous item |
| `Enter` | Select the focused item |
| `Escape` | Close the dialog (when used as `CommandDialog`) |

**ARIA:** The `CommandInput` is a combobox-role input; `cmdk` manages `aria-activedescendant` and live region announcements automatically. Give `CommandDialog` meaningful `title` and `description` props so screen reader users understand the palette's purpose when it opens.
